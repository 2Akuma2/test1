SmartID Option OPT_SMARTID_SMART_COMPONENT_CLASSES

Informationen zur Verwendung der Komponentenklasse in einer SmartID finden Sie in SmartID-Syntax für 'Klasse'.

Die in dieser Option enthaltenen Klassen sind: Accordion, Button, Calendar, CheckBox, Closer, ColorPicker, ComboBox, ComboButton, ComboField, Dialog, ErrorIndicator, Graphics, Icon, Indicator, Label, Link, List, Maximizer, Menu, MenuItem, Minimizer, OptionGroup, Panel:TitledPanel, PanelTitle, ProgressBar, RadioButton, Restore, Sizer, Slider, Spinner, Table, TabPanel, Text, TextArea, TextField, ToggleButton, ToolBarItem, ToolTip, Tree, TreeTable, Window.

Die enthaltenen Klassen können über ein SUT-Skript verändert werden, wobei alle gewünschten Klassen explizit angegeben werden müssen. Sie werden als kommaseparierte Liste in einem String spezifiziert, zum Beispiel 

rc.setOption(Options.OPT_SMARTID_SMART_COMPONENT_CLASSES, "Button,TextField")
Beispiel 44.1:  Setzen der Option OPT_SMARTID_SMART_COMPONENT_CLASSES

Nur die genannten Klassen (Button und TextField) brauchen nun nicht in der SmartID angegeben werden. Wenn Sie obige Liste also erweitern wollen, müssen Sie alle genannten Klassen plus die von Ihnen zusätzlich benötigten Klassen angeben.
