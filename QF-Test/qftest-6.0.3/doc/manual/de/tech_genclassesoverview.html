57.1
Übersicht der generischen Klassen

Jede generische Klasse hat bestimmte Eigenschaften, welche in diesem Kapitel aufgeführt werden. Zur leichteren Auffindbarkeit sind die Komponentenarten in folgenden Gruppen zusammengefasst. Details zu den Eigenschaften der jeweiligen Klasse finden Sie, indem Sie dem Link in der Spalte "Beschreibung" folgen.

Folgende Klassen stehen für Komponenten, die eine Eingabe bzw. einfache Anzeige von Text oder Bildern erlauben:
Klasse Beschreibung
ComboBox Besteht typischerweise aus einen Textfeld und einer Liste von auswählbaren Optionen. Siehe Abschnitt 57.2.8.
Graphics Zeigt eine Grafik oder ein Diagramm an. Klicks können Aktionen auslösen, müssen aber nicht. Siehe Abschnitt 57.2.12.
Icon Zeigt ein Bild an. Klicks können Aktionen auslösen, müssen aber nicht. Siehe Abschnitt 57.2.13.
Label Dient zur Anzeige einer Beschriftung. Klicks können Aktionen auslösen, müssen aber nicht. Siehe Abschnitt 57.2.16.
ProgressBar Zeigt den Fortschritt einer Aktion an. Siehe Abschnitt 57.2.28.
Slider Dient zur Auswahl eines Wertes mittels eines Schiebereglers. Siehe Abschnitt 57.2.32.
Spinner Dient zur Auswahl eines Wertes mittels Navigationsbuttons und eines Textfeldes. Siehe Abschnitt 57.2.34.
Text Beinhaltet einen Fließtext. Es ist keine Eingabe möglich. Siehe Abschnitt 57.2.43.
TextArea Ermöglicht mehrzeilige Textanzeigen bzw. -eingaben. Siehe Abschnitt 57.2.44.
TextField Ermöglicht einzeilige Textanzeigen bzw. -eingaben. Siehe Abschnitt 57.2.45.
Tabelle 57.1:  Generische Klassen für Eingabe und Anzeige

Folgende Klassen stehen für Komponenten, die eine direkte Interaktion, wie Klicks bieten und ggf. einen Status besitzen:
Klasse Beschreibung
Button Dient als Schaltfläche, die eine Aktion auslöst und keinen speziellen Status besitzt. Siehe Abschnitt 57.2.3.
CheckBox Besitzt einen bestimmten Status. Typischerweise sind diese an- und abhakbar. Siehe Abschnitt 57.2.6.
Closer Schließt andere Komponenten, wenn diese gedrückt wird, z.b. der 'X' Button eines Fensters. Siehe Abschnitt 57.2.7.
Link Dient zur Navigation auf einen anderen Bereich. Siehe Abschnitt 57.2.17.
Maximizer Maximiert einen Bereich, z.B. bei einem Fenster. Siehe Abschnitt 57.2.20.
MenuItem Wird in Menüs angezeigt. Ein Klick löst normalerweise eine Aktion aus oder wechselt den Zustand der Applikation. Siehe Abschnitt 57.2.22.
Minimizer Minimiert einen Bereich, z.B. bei einem Fenster. Siehe Abschnitt 57.2.23.
RadioButton Eine auswählbare Option, wird typischerweise bei Mehrfachauswahlen für eine bestimmte Option verwendet. Siehe Abschnitt 57.2.25.
Restore Stellt den Ursprungsstatus einer Komponente wieder her, z.B. bei einem Fenster. Siehe Abschnitt 57.2.26.
Sizer Verändert die Größe einer Komponente, z.B. bei einem Fenster. Siehe Abschnitt 57.2.31.
Thumb Stellt einen Schieberegler dar. Siehe Abschnitt 57.2.46.
ToggleButton Dient als Schaltfläche, die einen speziellen Status besitzt. Ein Klick kann auch eine Aktion auslösen. Siehe Abschnitt 57.2.47.
ToolBarItem Stellt ein klickbares Element in einer ToolBar dar. Siehe Abschnitt 57.2.49.
Tabelle 57.2:  Generische Klassen für Interaktionen

Hier erhalten Sie eine Übersicht, über alle Klassen, welche typischerweise für Container verwendet werden, und somit weitere Komponenten beinhalten:

Klasse Beschreibung
Accordion Dient als Navigator zwischen Komponenten. Komponenten können auf- bzw. zugeklappt werden. Siehe Abschnitt 57.2.1.
Calendar Dient zur Datumsauswahl. Siehe Abschnitt 57.2.4.
ColorPicker Dient zur Farbauswahl. Siehe Abschnitt 57.2.5.
FileChooser Dient zur Dateiauswahl. Besteht typischerweise aus einer Liste und mehreren Buttons. Siehe Abschnitt 57.2.11.
Indicator Zeigt eine Meldung nach einer Eingabe an. Typischerweise wird diese nach Eingabe in ein Textfeld erscheinen. Kann auch ein Icon beinhalten. Siehe Abschnitt 57.2.14.
Menu Beinhaltet mehrere auswählbare Menüeinträge. Siehe Abschnitt 57.2.21.
Popup Zeigt Komponenten an, wird typischerweise nach Klick auf einen Button angezeigt und hängt direkt bei diesem Button, z.B. bei einer ComboBox. Siehe Abschnitt 57.2.24.
Panel Beinhaltet weitere Komponenten. Wird zur Organisation eines Fensters verwendet. Siehe Abschnitt 57.2.27.
SplitPanel Beinhaltet weitere Komponenten, kann diese aber in eigenständig vergrößerbare Bereiche unterteilen. Siehe Abschnitt 57.2.35.
TabPanel Dient als Navigator zwischen Komponenten. Diese Komponenten liegen in separaten Tabs und es kann jeweils nur ein Tab sichtbar sein. Siehe Abschnitt 57.2.41.
ToolBar Stellt eine Werkzeugleiste dar. Beinhaltet typischerweise Menüeinträge oder wichtige Buttons für Aktionen. Siehe Abschnitt 57.2.48.
ToolTip Ist ein Fenster, welches als Hinweis geöffnet wird, z.B. wenn man mit der Maus darüber fährt. Siehe Abschnitt 57.2.50.
Window Stellt ein Fenster dar. Siehe Abschnitt 57.2.54.
Tabelle 57.3:  Generische Klassen für Container

Neben den Containerklassen für Komponenten gibt es noch Komponenten, die Daten in komplexeren Strukturen darstellen:

Klasse Beschreibung
List Zeigt mehrere Optionen an, welche auswählbar sein können. Siehe Abschnitt 57.2.18.
Table Dient zur Auswahl mehrerer Einträge, ist typischer mehrdimensional. Siehe Abschnitt 57.2.36.
TableFooter Fußzeile einer Tabelle. Siehe Abschnitt 57.2.38.
TableHeader Kopfzeile einer Tabelle. Siehe Abschnitt 57.2.39.
Tree Zeigt auswählbare Elemente in Baumdarstellung an. Dies dient zur Kategorisierung der Elemente. Siehe Abschnitt 57.2.51.
TreeTable Zeigt auswählbare Elemente in Baumdarstellung. Dies dient zur Kategorisierung der Elemente. Elemente werden typischerweise in mehreren Spalten dargestellt. Siehe Abschnitt 57.2.53.
Tabelle 57.4:  Generische Klassen für Datendarstellung

Für Daten selbst werden folgende Klassen verwendet:

Klasse Beschreibung
Item Element einer Liste. Kann auswählbar sein. Siehe Abschnitt 57.2.15.
TableCell Ein Wert in einer Tabelle, wird per Spalte und Zeile identifiziert. Siehe Abschnitt 57.2.37.
TableHeaderCell Spaltenname einer Tabelle. Siehe Abschnitt 57.2.40.
TableRow Zeile einer Tabelle. Siehe Abschnitt 57.2.42.
TreeNode Ein Element eines Baumes. Siehe Abschnitt 57.2.52.
Tabelle 57.5:  Generische Klassen für Daten

Neben Klassen für Eingabe, Interaktion, Daten oder Gruppierung gibt es noch eine Reihe von Klassen, welche nur in einem bestimmten Kontext auftreten:

Klasse Beschreibung
BusyPane Wird über andere Komponenten gelegt, um diese zu sperren. Siehe Abschnitt 57.2.2.
Divider Trennt Bereiche mehrerer Komponenten voneinander ab und ist verschiebbar. Siehe Abschnitt 57.2.9.
Expander Dient zum Auf- und Zuklappen von Komponenten, z.B. bei Baumknoten. Siehe Abschnitt 57.2.10.
LoadingComponent Dient zur Anzeige, dass Ihre Anwendung gerade beschäftigt ist, z.B. um etwas einzulesen. Siehe Abschnitt 57.2.19.
ScrollBar Bereich, der für das Scrollen von Komponenten benutzt wird. Beinhaltet typischerweise einen Schieberegler. Siehe Abschnitt 57.2.29.
Separator Trennt Bereiche mehrerer Komponenten voneinander ab und ist nicht verschiebbar. Siehe Abschnitt 57.2.30.
Spacer Stellt einen Einrückeffekt dar, typischerweise bei Baumknoten. Siehe Abschnitt 57.2.33.
